# AGENTS.md

Guidance for AI coding agents working on the `modular-sticky` repository.

## Architecture Overview

This is a **CommonJS Electron application** (no TypeScript compilation, no bundler, no transpile step). All source files are plain `.js` that Electron runs directly. There are exactly 4 source files:

| File          | Process  | Role                                              |
| ------------- | -------- | ------------------------------------------------- |
| `main.js`     | Main     | Window lifecycle, IPC handlers, JSON store        |
| `preload.js`  | Preload  | Context bridge -- exposes limited API to renderer |
| `renderer.js` | Renderer | UI logic, auto-save, color contrast               |
| `note.html`   | Renderer | Single HTML template for all note windows         |

## Process Architecture

- **Main process** (`main.js`): Owns `noteWindows` (a `Map<id, BrowserWindow>`). One `BrowserWindow` per note. Windows are frameless (`frame: false`), always-on-top capable.
- **Preload** (`preload.js`): Runs with `contextIsolation: true`, `nodeIntegration: false`. Exposes `window.electronAPI` via `contextBridge.exposeInMainWorld`.
- **Renderer** (`renderer.js` + `note.html`): No direct Node access. All Electron interaction goes through `window.electronAPI`.

## IPC Channels

These are the **actual implemented channels** from the current source code.

### Renderer to Main (invoke/handle)

| Channel                | Arguments                  | Returns               | Purpose                          |
| ---------------------- | -------------------------- | --------------------- | -------------------------------- |
| `create-note`          | none                       | void                  | Spawn a new note window          |
| `close-note`           | `id: string`               | void                  | Close and delete a specific note |
| `save-note-content`    | `{ id, content }`          | void                  | Persist note body text           |
| `update-note-color`    | `{ id, color: hexString }` | void                  | Change note background color     |
| `minimize-note`        | `id: string`               | void                  | Minimize a note window           |
| `toggle-always-on-top` | `id: string`               | `boolean` (new state) | Toggle window always-on-top      |

### Main to Renderer (send/on)

| Channel        | Payload                  | Trigger                          |
| -------------- | ------------------------ | -------------------------------- |
| `load-note`    | `{ id, content, color }` | Fired after `did-finish-load`    |
| `request-save` | none                     | Fired on `before-quit` app event |

## Data Schema

Notes are stored in `<userData>/notes.json` as a plain JSON array:

```json
[
  {
    "id": "uuid-v4-string",
    "content": "plain text content",
    "color": "#ffeb3b",
    "x": 100,
    "y": 100,
    "width": 300,
    "height": 300
  }
]
```

- `id`: generated by `uuid` package (`uuidv4()`)
- `color`: one of the 7 defined hex values (see below)
- `x`, `y`: window position; new notes cascade by `existingCount * 30`
- `width`, `height`: window size; default `300 x 300`
- `content`: raw text string (no HTML, no markdown -- plain `textarea` value)

## Color Palette

The 7 valid note colors (from `note.html` swatches):

```
#ffeb3b  (yellow -- default)
#ff9800  (orange)
#4caf50  (green)
#2196f3  (blue)
#e91e63  (pink)
#9c27b0  (purple)
#ffffff  (white)
```

Text color adapts based on luminance: `(r*299 + g*587 + b*114) / 1000`. Threshold is 128 -- above = dark text (`#333`), below = light text (`#fff`).

## Build System

```bash
npm start                  # Run in dev (Electron directly)
npm run dist:current       # Build for current platform
npm run dist:linux         # AppImage + deb + rpm + snap + tar.xz
npm run dist:mac           # dmg + zip + pkg (x64 + arm64 + universal)
npm run dist:win           # nsis + msi + zip + portable
npm run build-clean        # Full build via scripts/compile-build-dist.sh
npm run bloat-check        # Analyze bundle size
npm run temp-clean         # Clean build-temp/ directory
```

Build output goes to `dist/`. Resources (icons) come from `build-resources/`. The build script sets `ELECTRON_BUILDER_PARALLELISM=18` and uses `build-temp/` as TMPDIR.

**Linux sandbox fix**: Electron requires unprivileged user namespaces or the no-sandbox flag:

```bash
sudo sysctl -w kernel.unprivileged_userns_clone=1
# or launch with: electron . --no-sandbox
```

## Code Style

- CommonJS only: `require()` / `module.exports`. No `import`/`export`.
- No classes -- plain functions and object literals.
- ESLint + Prettier enforced. Run `npm run lint` before committing.
- TypeScript is present for **type checking only** (`tsc --noEmit`). No `.ts` source files exist.
- Auto-save uses a 1000ms debounce via `setTimeout`/`clearTimeout` on the `input` event.

## Security Rules

- Never enable `nodeIntegration: true` in any `BrowserWindow`.
- Never disable `contextIsolation`.
- All new IPC channels must go through `contextBridge` -- never expose raw `ipcRenderer`.
- Do not load external URLs in any `BrowserWindow`. All content is local.
- Validate IPC payloads in `ipcMain.handle` before using them.

## What Does NOT Exist (Do Not Reference)

The following features are **not implemented** despite appearing in some older documentation:

- Search / note filtering
- Rich text editing (storage is a plain `<textarea>`, not a rich editor)
- Dark mode / theme switching
- Cloud sync or auto-updater
- Categories or tags
- `src/` directory -- there is no such directory
- Any external database -- storage is a single JSON file in `userData`
- Touch Bar, system tray, or auto-startup integrations

Do not add these features without explicit instruction. Do not reference them in documentation or code comments.

## Common Tasks

**Add a new note property**: Update the data schema in `main.js` (store read/write), add the IPC channel in both `main.js` and `preload.js`, handle it in `renderer.js`.

**Add a color**: Add a swatch `<div>` in `note.html` with the hex value as `data-color`. The contrast logic in `renderer.js` handles text color automatically.

**Add a keyboard shortcut**: Add `addEventListener('keydown', ...)` in `renderer.js`. Check `event.key` / `event.ctrlKey` / `event.metaKey`.

**Debug IPC**: `console.log` in `main.js` appears in the terminal. `console.log` in `renderer.js` appears in the DevTools console (open with `webContents.openDevTools()` in main, or `Ctrl+Shift+I` on a note window if devtools are enabled).
